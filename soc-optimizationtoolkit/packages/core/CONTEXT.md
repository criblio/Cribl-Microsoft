# CONTEXT: @soc/core

Purpose: pure domain logic and the port interfaces both app shells bind against.

Boundaries: imports nothing from ui or apps. Zero IO, zero fetch, zero React. Everything here must be unit-testable with plain vitest and fakes.

Invariants:
- Port interfaces are the only seam between shared code and the two shells (cloud platform adapters vs local Node host).
- DCR/DCE name generation must reproduce legacy output exactly (compatibility contract; characterization tests mandatory).
- assets/cribl-openapi.json is the vendored Cribl API spec (pinned per Cribl version); typed API client shapes are written against it.

Status: Phase 1 foundation in place. src/ports/ defines the six port interfaces (SecretsStore, UserContext, ArtifactSink, JobStore, AzureManagement, CriblClient) plus shared HTTP types; src/testing/ provides in-memory fakes for all six; src/domain/dcr-naming/ and src/domain/schema-mapping/ implement the legacy compatibility contracts with characterization tests pinned to legacy vectors/fixtures. src/usecases/azure-discovery/ (porting-plan Unit 2) provides listSubscriptions (Enabled-only, pinned), listWorkspaces, listResourceGroupChoices (verbatim deriveResourceGroupsFromWorkspaces fallback), createResourceGroup, createWorkspace (attempt-bounded provisioning poll), enableSentinel (workspace's ACTUAL location - legacy eastus bug fixed + pinned), listAllPages (nextLink pagination via the OPTIONAL AzureManagement.requestUrl, single-page fallback without it), and commitTargetScope (merge through azure-profiles + connection-invalidation). All are re-exported from src/index.ts (including AzureManagementUrlRequest from ports/). Remaining domain modules (pipeline-generation, field-matcher, pack-assembly, reduction-rules, sample-parsing) and the GraphClient port land per the implementation roadmap.

Unit 4 (porting-plan): src/domain/option-forms/ - deployment and naming options as typed form DATA mined from the legacy param-forms.ts (azure form NOT ported - superseded by azure-profiles). OperationOptions + CriblOptions with DEFAULTS; OptionFormField descriptor lists whose descriptions carry the legacy operational knowledge (30/64-char DCR name limits, Cribl 4.14+ for Direct, AMPLS implications); tolerant per-field parse codecs (parseAppOptions/parseOperationOptions/parseCriblOptions, junk -> that field's default) plus canonical serializeAppOptions; validateOptions REJECTS non-numeric number input with named per-field errors (DECISION pinned by test: the legacy `Number(val)||0` silently persisted 0); formValuesToOptions/optionsToFormValues for descriptor-driven renderers; immutable getNestedValue/setNestedValue; applyOptionsPatch - the MERGE-PRESERVING save (unmanaged keys in the stored blob survive, pinned by test); destinationIdFromOptions (pinned against defaultSentinelDestinationId so the duplicated sanitize rule cannot drift).
